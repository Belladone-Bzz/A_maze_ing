	**“A labyrinth is not a place to be lost, but a path to be found.”**


The aim of this project is to create a programme capable of generating a  maze and then solving it by identifying the shortest path between the entrance and the exit. Maze generation is a great source of complexity in term projects.

Before starting this project, we looked into practical applications for generating and solving mazes in computer science. Maze generation can be extremely useful in many fields: video games, robotics, geolocation and transport, art...

This project allowed us to greatly improve the following skills:

- Component conception
- Graph generation
- Package management
- Recursion
- 2D rendering
- Object-Oriented programming
- Data Serialization
- Configuration Management
- Packaging & Distribution
- Graph traversal
- Simple pathfinding
- Functional Documentation




Un **labyrinthe** est un tracé sinueux, muni ou non d'embranchements, d'impasses et de fausses pistes, destiné à égarer ou à ralentir celui qui cherche à s'y déplacer. Ce motif, apparu dès la Préhistoire, se retrouve dans de nombreuses civilisations sous des formes diverses. Dans la mythologie grecque, le labyrinthe est une série de galeries construites par l'architecte Dédale pour enfermer le Minotaure.

Au sens figuré, le labyrinthe correspond à un système complexe, tortueux, qui peut être concret (architecture urbanisme, jardins...) ou abstrait (structures, façons de penser…) et où il est fréquent de se perdre, faute de saisir la globalité du cheminement.

Les labyrinthe peuvent etre separes en deux catégories :
- Les labyrinthes dits « parfaits » où chaque cellule est reliée à toutes les autres et, ce, de manière unique.
- Les labyrinthes dits « imparfaits » qui sont tous les labyrinthes qui ne sont pas parfaits (ils peuvent donc contenir des boucles, des îlots ou des cellules inaccessibles).
